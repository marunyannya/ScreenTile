Firebaseプロジェクトの作成

ScreenTile はGoogleのクラウドサービス Firebase を使ってデータを管理します。まずはプロジェクトを作成してください。

  1. Firebase Console にアクセスし、Googleアカウントでログインします。
  2. 「プロジェクトを作成」をクリックします。
  3. プロジェクト名を入力します（例：ScreenTile）。
  4. Google アナリティクスの設定は任意です。不要であれば無効にして進んでください。
  5. 「プロジェクトを作成」ボタンを押し、作成完了まで待ちます。
Firebaseの無料プラン（Sparkプラン）でScreenTileは動作します。クレジットカードの登録は不要です。
2

Authentication の設定

ScreenTile は「匿名認証」を使って教員・学生を区別します。以下の手順で有効化してください。

  1. Firebase Console の左メニューから 「構築」→「Authentication」 を選択します。
  2. 「始める」をクリックします。
  3. 「Sign-in method」タブを開きます。
  4. 「匿名」をクリックし、「有効にする」 をオンにして保存します。
「Google」や「メール／パスワード」などの他のサインイン方法は有効にしなくて構いません。
3

Firestore Database の作成

  1. Firebase Console の左メニューから 「構築」→「Firestore Database」 を選択します。
  2. 「データベースを作成」をクリックします。
  3. ロケーションは 「asia-northeast1（東京）」 を推奨します。
  4. セキュリティルールの初期設定は 「本番環境モード」 を選択します（後でルールを設定します）。
  5. 「作成」をクリックして完了です。
4

Firestoreセキュリティルールの設定

不正なアクセスを防ぐため、以下のセキュリティルールを設定してください。

  1. Firestore Database のページで 「ルール」タブ を開きます。
  2. 既存の内容をすべて削除し、以下のルールをコピー＆ペーストします。
  3. 「公開」ボタンをクリックして保存します。
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    function isAuthenticated() {
      return request.auth != null;
    }

    // 1. ルーム情報
    match /artifacts/{appId}/public/data/rooms/{roomId} {
      // 認証済みユーザーなら誰でも読める（学生がルーム存在確認に使う）
      allow read: if isAuthenticated();
      // ルームの新規作成：hostId は自分のUIDに限定
      allow create: if isAuthenticated()
                    && request.resource.data.hostId == request.auth.uid;
      // 更新・削除：作成者本人のみ
      allow update, delete: if isAuthenticated()
                            && resource.data.hostId == request.auth.uid;
    }

    // 2. 学生情報
    match /artifacts/{appId}/public/data/students/{studentId} {
      // 認証済みユーザーなら誰でも読める（教員がタイル表示に使う）
      allow read: if isAuthenticated();
      // 登録・更新・削除：自分のドキュメントのみ
      allow create, update, delete: if isAuthenticated()
                                    && request.auth.uid == studentId;
    }
  }
}
このルールでは「Firebaseに認証済みのユーザーのみ」がデータを読み書きできます。認証なしの第三者はアクセスできません。
5

firebase.json（設定ファイル）の作成と公開

① 設定値の取得

  1. Firebase Console のプロジェクトトップページを開きます。
  2. 「アプリを追加」から 「ウェブ」（</>） を選択します。
  3. アプリのニックネームを入力し（例：screentile-web）、「アプリを登録」をクリックします。
  4. 表示された firebaseConfig の中身をメモします。

② firebase.json の作成

メモした値を使って、以下の形式で firebase.json ファイルを作成します。キーもダブルクォートで囲むことに注意してください。

{
  "apiKey": "AIzaSy...",
  "authDomain": "yourproject.firebaseapp.com",
  "projectId": "yourproject",
  "storageBucket": "yourproject.firebasestorage.app",
  "messagingSenderId": "123456789",
  "appId": "1:123456789:web:abcdef"
}
JavaScript のオブジェクト記法（キーにクォートなし）では読み込めません。必ず正しいJSON形式で記述してください。

③ ファイルの公開（GitHub Pages を使う場合）

  1. GitHubにリポジトリを作成します（例：ScreenTile）。
  2. 作成した firebase.json をリポジトリにアップロードします。
  3. リポジトリの「Settings」→「Pages」から GitHub Pages を有効にします。
  4. 公開後のURLは以下の形式になります。
https://（GitHubユーザー名）.github.io/（リポジトリ名）/firebase.json
GitHub Pages はデフォルトでCORSを許可しているため、追加設定なしにファイルを読み込めます。他のホスティングサービスを使う場合は、CORS設定が必要になることがあります。
6

ScreenTileのファイルを公開する

ScreenTile は以下の3つのファイルで構成されています。すべて同じリポジトリ（Step 5 と同じ場所）に配置することを推奨します。

teacher.html　← 教員ページ（URLは教員のみに共有）

student.html　← 学生ページ（こちらを学生に共有）

howto.html　　← この使い方ガイド

firebase.json　← Firebase設定ファイル

teacher.html のURLは学生に知らせないでください。URLを知られると、学生が教員ページにアクセスできてしまいます。
7

URLの指定方法とアクセス手順

教員のアクセス手順

  1. 以下の形式でURLを指定してブラウザでアクセスします。
https://（公開先）/teacher.html?config=https://（公開先）/firebase.json

例：

https://example.github.io/ScreenTile/teacher.html?config=https://example.github.io/ScreenTile/firebase.json
  1. ルームナンバー（任意の半角英数字）・パスワード・講義名を入力して入室します。
  2. 学生には ルームナンバーのみ を伝えます（パスワードは教員間での共有に使います）。

学生のアクセス手順

  1. 以下の形式でURLを指定してブラウザでアクセスします。
https://（公開先）/student.html?config=https://（公開先）/firebase.json
  1. 教員から伝えられたルームナンバーと氏名を入力して入室します。
  2. 学籍番号は任意入力です（半角英数字）。
  3. 入室後、「画面共有を開始する」ボタンから画面共有を開始します。
毎回URLを入力させるのが手間な場合は、QRコードや短縮URLサービス（例：bit.ly）を活用すると便利です。

複数教員でルームを共有する場合

同じルームナンバーとパスワードを教員間で共有することで、複数の教員が同じルームに入室できます。学生のタイル画面は全員で同時に閲覧できます。
